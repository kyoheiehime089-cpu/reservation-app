# friends 無料体験予約フォーム

friends のセミパーソナルジム無料体験に申し込むための、GitHub Pages向け静的予約リクエストフォームです。

このフォームは **予約確定フォームではありません**。お客様が第1〜第3希望日時を送信し、管理者がFormspreeの投稿履歴・通知メールを確認してから、メール返信で正式に予約を確定します。

## 公開URL

- GitHub Pages: <https://kyoheiehime089-cpu.github.io/reservation-app/>
- GitHub Pages の Source は **GitHub Actions** です。`Deploy from a branch` へ変更する必要はありません。

## この予約フォームの使い方

### お客様

1. 公開URLを開きます。
2. お名前、電話番号、メールアドレスを入力します。
3. 第1希望日時、第2希望日時、第3希望日時をすべて選択します。
4. 必要に応じて連絡事項を入力します。
5. 「予約リクエストを送信する」を押します。
6. 送信成功時は次の完了メッセージが表示されます。

```txt
無料体験の予約リクエストを受け付けました。
確認後、入力いただいたメールアドレス宛に予約確定のご連絡をお送りします。
```

### 管理者

1. Formspree管理画面を開きます。
2. 対象フォーム `https://formspree.io/f/xkoyzpdv` の投稿履歴（Submissions）を確認します。
3. 通知メールまたは投稿履歴で、送信日時、お名前、電話番号、メールアドレス、第1希望日時、第2希望日時、第3希望日時、連絡事項を確認します。
4. 入力されたメールアドレスへ、管理者から予約確定メールを返信します。

## Formspreeを使う理由

GitHub Pagesは静的サイトのため、サーバー側で予約データ保存やメール送信を直接行えません。Formspreeを使うことで、静的HTML/CSS/JavaScriptだけで次のことができます。

- フォーム送信内容をFormspreeの投稿履歴に保存する
- 管理者宛に通知メールを送る
- `_replyto` に予約者のメールアドレスを設定し、管理者が返信しやすくする
- Formspree側の機能で自動返信メールを設定する

## Formspree設定

- Formspree送信先URL: `https://formspree.io/f/xkoyzpdv`
- `FORM_ENDPOINT` の設定場所: `src/main.js` の先頭

```js
const FORM_ENDPOINT = "https://formspree.io/f/xkoyzpdv";
```

FormspreeのマスターAPIキー、読み取り専用APIキー、その他のAPIキーは使いません。公開リポジトリへAPIキーをコミットしないでください。

## Formspreeへ送信されるデータ

送信成功時、`fetch` でFormspreeへPOSTします。送信データには次の項目が含まれます。

- `_subject`: `【friends】無料体験予約リクエストが届きました`
- `_replyto`: 予約者が入力したメールアドレス
- 送信日時
- お名前
- 電話番号
- メールアドレス
- 第1希望日時
- 第2希望日時
- 第3希望日時
- 連絡事項
- フォーム種別
- 管理者確認メモ

これにより、管理者はFormspreeの投稿履歴と通知メールから予約者へ予約確定メールを返信できます。

## 送信テストの方法

1. ローカルまたはGitHub Pagesでトップページを開きます。
2. 必須項目をすべて入力します。
3. 第1〜第3希望日時をすべて別の日時で選択します。
4. 「予約リクエストを送信する」を押します。
5. 送信中はボタンが無効化され、「送信中です…」に変わることを確認します。
6. 送信成功時だけ完了画面に切り替わることを確認します。
7. Formspree管理画面の投稿履歴と通知メールで、予約内容が届いていることを確認します。
8. 送信に失敗した場合は、画面上に「送信に失敗しました。時間をおいて再度お試しください。」と表示され、完了画面に切り替わりません。

## お客様への自動返信メール設定

GitHub Pagesだけではメール送信ができないため、自動返信メールはコードでは実装しません。Formspreeの自動返信、ワークフロー、テンプレート機能で設定してください。

### 設定手順

1. Formspreeにログインします。
2. フォーム `xkoyzpdv` を開きます。
3. 自動返信、ワークフロー、またはメールテンプレートの設定画面を開きます。
4. 宛先を予約者のメールアドレス項目に設定します。
5. 件名と本文を下記の内容で設定します。
6. テスト送信し、予約者のメールアドレスに自動返信が届くことを確認します。

### 自動返信メールの件名

```txt

```

## admin.htmlについて

`admin.html` は簡易確認用です。同じブラウザで送信に成功した予約リクエストを `localStorage` から一覧表示できます。

ただし、`localStorage` は別端末・別ブラウザと共有されません。本番の予約保存場所はFormspree側です。正式な確認は必ずFormspree管理画面の投稿履歴と通知メールで行ってください。

表示項目は次の通りです。

- 送信日時
- お名前
- 電話番号
- メールアドレス
- 第1希望日時
- 第2希望日時
- 第3希望日時
- 連絡事項

## Googleカレンダー連携について

このアプリではGoogleカレンダー追加ボタン、カレンダー登録URL生成処理、カレンダー追加画面は不要にしました。正式な予約確定は、管理者が希望日時を確認してお客様へメール返信する運用です。

## ローカルで起動する方法

```bash
npm install
npm run dev
```

表示されたURLをブラウザで開きます。通常は `http://localhost:5173/` です。簡易確認画面は `http://localhost:5173/admin.html` で開けます。

## ビルド

```bash
npm run build
```

ビルドでは、GitHub Pages公開用に静的ファイルを `dist/` へコピーします。
